Search Syntax

The search box filters links by name and target URL. The query supports optional field prefixes and anchor characters.

Field Prefixes

By default the query is matched against both the link name and the target URL. A prefix restricts the match to a single field:

PrefixSearchesExample
name: or n: Link name only name:docs
target: or t: Target URL only t:github.com
(none) Both name and target docs

Anchors

By default the query matches anywhere in the field (substring match). Use ^ and $ to anchor the match:

SyntaxMeaningExample
^foo Must start with foo name:^go- matches go-docs but not my-go-link
foo$ Must end with foo name:docs$ matches go-docs but not docs-old
^foo$ Exact match name:^go-docs$ matches only go-docs

Examples

QueryFinds
designLinks whose name or target contains design
n:^engLinks whose name starts with eng
t:github.com/myorgLinks pointing into github.com/myorg
name:^wiki$The link named exactly wiki
